
# üìö Book Review API

A RESTful API built with **Node.js**, **Express**, **MongoDB**, and **JWT** for user authentication. This API allows users to manage books and reviews.

---

## üöÄ Project Setup

### 1. Clone and install dependencies
```bash
git clone https://github.com/yourusername/book-review-api.git
cd book-review-api
npm init -y
npm install
```

### 2. Create a `.env` File in the Root Directory

```env

MONGO_URI=mongodb://localhost:27017/bookreviews
JWT_SECRET=your_secret_key

üí° Ensure MongoDB is running locally or use MongoDB Atlas connection string for production.
```

### 3. Running the App Locally

```

The API will run on: http://localhost:5000
```

```

### 5. Book APIs
### 5.1 Create Books ‚Äì POST /api/books
```API Url: http:localhost:5000/api/books```
```
Headers:
Authorization: your_jwt_token

Body: 
{
  "title": "The Alchemist",
  "author": "Paulo Coelho",
  "genre": "Fiction"
}

```

### 5.2 GET ALL Books ‚Äì GET /api/books
```API Url: http:localhost:5000/api/books```
```
Query Params:
1. author=paulo
2. genre=fiction
3. page=1&limit=10

Response is paginated as per 10 records per page
```

### 5.3 Search Books ‚Äì GET /api/books/search?q=queryname
```API Url: http:localhost:5000/api/search?q=queryname```
```
Query Params:
1. author=paulo
2. title=fiction

Filters on the basis of author or title provided in query params
```

```API Url: http:localhost:5000/api/books/id```

### 6. Review APIs

```
Headers:

  "rating": 4,

### 7. üí° Decisions & Assumptions

```
‚úÖ JWT Authentication secures private routes.

üö´ One user can review a book only once (enforced with a compound index on book + user using MongoDB).

üõ°Ô∏è Users can only update/delete their own reviews.

üìÑ Reviews and book listings support pagination.

üîç Search supports partial matches on title or author.
```
### 8. Database Schema
### 8.1 User Model

```
{
  username: String, // required, unique
  email: String,    // required, unique
  password: String  // hashed using bcrypt
}

```

### 8.2 Book Model

```
{
  title: String,        // required
  author: String,       // required
  genre: String,
}

```

### 8.3 Review Model

```
{
  book: ObjectId,   // reference to Book
  user: ObjectId,   // reference to User
  rating: Number,   // min: 1, max: 5
  comment: String
}

```
```üìå Compound Unique Index: { book: 1, user: 1 } to prevent duplicate reviews.```
